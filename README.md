# 插件仓库

收录并分发 SPlayer-Next 的插件。

> [!IMPORTANT]
> 编写插件前请完整阅读 **[插件开发文档](https://splayer-next.imsyy.top/plugins/)**。脚本头字段、可用 API、编写规范等均以该文档为准。

---

## 提交流程

### 1. 准备插件

按[插件开发文档](https://splayer-next.imsyy.top/plugins/)编写脚本。脚本头需声明 `@id`、`@name`、`@version`、`@type`、`@apiLevel` 等字段，其中 `@id` 全局唯一、上架后不可更改。

### 2. 提交

新建 Issue，按用途选择「新建插件」或「更新插件」模板，填写 `@id`，并上传 `.js` 文件或粘贴脚本 raw 直链。

机器人会自动拉取脚本并进行校验，将结果回贴于 Issue：

- **通过**：自动创建 Pull Request，转入审核。
- **未通过**：回贴具体原因；按提示修正后编辑 Issue 将自动重新校验。

### 3. 等待审核

维护者将对你的插件进行人工审核。审核通过后插件即收录，并出现在客户端的插件市场；若被驳回，请按审核意见修改后重新提交。

## 更新插件

提高脚本头部的 `@version`，使用「更新插件」模板重新提交，`@id` 保持不变。审核合并后，客户端会检测到新版本并提示用户一键更新。
